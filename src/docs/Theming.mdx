# Theming

Abadikan DS uses a three-tier CSS custom property system so you can brand
the library without touching any component code.

## Token tiers

| Tier | Example | Purpose |
|------|---------|---------|
| **Primitive** | `--blue-500: oklch(0.55 0.18 250)` | Raw palette values |
| **Semantic** | `--color-primary: var(--blue-500)` | Intent-based aliases |
| **Component** | `--btn-bg: var(--color-primary)` | Per-component overrides |

Override at any tier in your own CSS — changes cascade downward automatically.

## Quick override (CSS)

The simplest way to re-brand is to override semantic tokens in your global stylesheet:

```css
:root {
  --color-primary:       oklch(0.55 0.22 160);  /* green */
  --color-primary-hover: oklch(0.50 0.22 160);
  --color-primary-light: oklch(0.93 0.08 160);
  --radius-md: 0.75rem;                          /* rounder corners */
}
```

## `createTheme` / `applyTheme` API

For multi-brand or per-tenant theming, use the JS helpers:

```ts
import { createTheme, applyTheme } from '@abadikan/ds'

// createTheme returns a flat Record<string, string> of CSS vars
const greenTheme = createTheme({
  primary:      'oklch(0.55 0.22 160)',
  primaryHover: 'oklch(0.50 0.22 160)',
  primaryLight: 'oklch(0.93 0.08 160)',
  radiusScale:  'soft',
  fontUi:       '"DM Sans", sans-serif',
})

// applyTheme injects vars onto a target element (default: document.documentElement)
// and returns a cleanup function to remove them
const cleanup = applyTheme(greenTheme)

// Later — remove the theme
cleanup()
```

### Scoped theming

Pass a container element as the second argument to scope the theme:

```ts
const el = document.querySelector('#tenant-widget')
applyTheme({ primary: 'oklch(0.55 0.20 30)' }, el)
```

### `radiusScale` values

| Value | Effect |
|-------|--------|
| `sharp` | `--radius-md: 0.125rem` |
| `default` | `--radius-md: 0.375rem` *(library default)* |
| `soft` | `--radius-md: 0.75rem` |
| `round` | `--radius-md: 1rem` |
| `pill` | `--radius-md: 9999px` |

## Motion tokens

Reduce animation by overriding duration tokens:

```css
/* Respect prefers-reduced-motion */
@media (prefers-reduced-motion: reduce) {
  :root {
    --duration-enter:  0ms;
    --duration-exit:   0ms;
    --duration-expand: 0ms;
  }
}
```

The library does **not** apply `prefers-reduced-motion` automatically — add the
override above in your global CSS to opt in.

## Elevation tokens

Shadows are aliased through elevation tokens so you can swap shadow styles globally:

```css
:root {
  --elevation-1: 0 1px 3px rgb(0 0 0 / 0.08), 0 1px 2px rgb(0 0 0 / 0.04);
  --elevation-2: 0 4px 8px rgb(0 0 0 / 0.10), 0 2px 4px rgb(0 0 0 / 0.06);
}
```

## Dark mode

Set `data-theme="dark"` on `<html>` or any container. The library switches all
semantic tokens automatically via CSS `:root[data-theme='dark']` overrides.

```ts
// Programmatic toggle
document.documentElement.dataset.theme = isDark ? 'light' : 'dark'
```
